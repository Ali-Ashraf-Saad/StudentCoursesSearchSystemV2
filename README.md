   # منصة المقررات الدراسية الذكية (Students Courses Portal)

   منصة رقمية متكاملة صُممت لطلاب الكلية بهدف تسهيل حياتهم الأكاديمية. تتيح المنصة
   تجربة سلسة تشمل:

   - **الاستعلام السريع**: عن المواد الدراسية المقررة وتفاصيل الامتحانات (رقم
   اللجنة، المكان، التاريخ، والوقت).
   - **تتبع المقررات**: للفرق الدراسية الأربع بجميع أقسامها لمعرفة الخطة
   الأكاديمية.
   - **حاسبة GPA**: لحساب المعدل الفصلي والتراكمي بدقة وبأكثر من طريقة مرنة.
   - **دليل الطالب**: توفير إجابات لأشهر الأسئلة الشائعة مع تجميع لأهم الروابط
   الأكاديمية.
   - **لوحة تحكم (Dashboard)**: إحصائيات متقدمة لمراقبة نشاط النظام وحركة الزوار
   بشكل لحظي.

   تعتمد المنصة على نظام معالجة بيانات شبه آلي يقوم بتحويل الملفات النصية الخام إلى قواعد بيانات JSON خفيفة وسريعة لضمان أفضل أداء.

   **الموقع المباشر:** https://studentscourses.gt.tc

   ---

   ## الميزات الرئيسية

   - **استعلام ذكي عن الامتحانات:** بحث فوري بالاسم أو الرقم الأكاديمي مع دعم الأخطاء الإملائية والتشكيل.
   - **تتبع المقررات (الخطة الدراسية):** واجهة تفاعلية لمعرفة المواد المنجزة والمتبقية، والمتطلب السابق والمواد التي يفتحها كل مقرر  .
   - **حاسبة GPA احترافية:** تدعم طرق حساب متعددة (بالمعدل الفصلي، بالتقديرات، أو بالدرجات المئوية) مع رسوم بيانية لمستوى الطالب.
   - **لوحة إحصائيات (Dashboard):** تتبع لحظي لعدد الزوار والنشاط مقسمة بالساعات والأيام والشهور.
   - **معالجة آلية للبيانات:** سكريبتات Python متكاملة لتحويل وتجهيز البيانات بضغطة زر واحدة.

   ---

   ## هيكل المشروع وشرح الملفات

   ينقسم المشروع إلى واجهة أمامية (Frontend)، الواجهة الخلفية (PHP Backend)، ونظام معالجة بيانات (Python Data Pipeline).

   ```
   ├── index.php                # الصفحة الرئيسية — واجهة البحث عن الامتحانات
   ├── search.php               # API البحث عن الطلاب والمقررات
   ├── stats.php                # لوحة الإحصائيات والتحليلات
   ├── courses.html             # صفحة المقررات الدراسية وتتبع المقررات
   ├── gpa.html                 # حاسبة المعدل التراكمي والفصلي
   ├── qa.html                  # صفحة الأسئلة الشائعة والروابط الهامة
   ├── tour-guide.js            # سكريبت الجولة التعريفية التفاعلية للمستخدمين
   ├── dashboardOld.html        # لوحة التحليلات القديمة (محفوظة كمرجع)
   ├── .htaccess                # إعدادات خادم Apache (لإخفاء الامتدادات والتوجيه)
   ├── router.php               # موجه الطلبات (Router) لتشغيل خادم PHP المحلي
   │
   ├── counterFiles/            # نظام تتبع الزيارات والإحصائيات
   │   ├── counter.php             # API تسجيل وقراءة الزيارات
   │   ├── counter-lib.php         # مكتبة دوال العداد الأساسية
   │   ├── logs/                   # سجلات JSONL لكل صفحة
   │   └── stats_cache/            # كاش مؤقت تولّده stats.php لتسريع الاستجابة
   │
   ├── txtData/                 # ① البيانات الخام (يُعدَّل يدوياً كل فصل)
   │   ├── CS.txt ، IT.txt ، IS.txt ، gen.txt # قوائم الطلاب والمواد لكل قسم
   │   ├── rooms.txt               # عدد الطلاب واللجان لكل مادة يُولَّد بـ rooms.py
   │   └── subjectsName.txt        # قائمة لأسماء المواد يُولَّد بـ subjectName.py
   │
   ├── codePyData/              # ② سكريبتات المعالجة
   │   ├── RUN.sh                  # سكريبت التشغيل المجمع (يُشغل المعالجة بضغطة واحدة)
   │   ├── foldersOutput.py        # يحوّل الملفات الخام لمجلدات لجان مقسمة
   │   ├── jsonOutput.py           # يجمّع المجلدات ويُنتج ملفات JSON النهائية
   │   └── ...                     # (سكريبتات مساعدة أخرى لتنظيف وتوليد البيانات)
   │
   ├── foldersData/             # ③ مخرجات وسيطة (تُولَّد آلياً من foldersOutput.py)
   │   ├── CS/ ، IT/ ، IS/ ، gen/  # اللجان مقسمة في ملفات نصية صغيرة
   │
   ├── data/                    # ④ قاعدة البيانات النهائية (JSON) (لا تُعدََّل ملفاته يدوياً)
   │   ├── students.json           # بيانات الطلاب والمقررات المسجلين بها
   │   ├── courses.json            # تفاصيل المقررات (الاسم، اليوم، التاريخ، الفترة، القاعات)
   │   ├── exams.json              # بيانات لجان الامتحانات المفصلة
   │   ├── rooms.json              # قائمة القاعات المستخدمة
   │   └── subjects.json           # شجرة المواد ومتطلباتها السابقة (Prerequisites)
   │
   ├── getInfo/                 # أدوات سطر أوامر (CLI) للمطورين للاختبار
   │   ├── getStdJson.py           # اختبار البحث في ملفات JSON
   │   └── getStdfoldersData.py    # اختبار البحث في المجلدات الوسيطة
   │
   └── images/                  # (أيقونات الموقع، شعار الكلية، وصور)
   ```

   ### 1. مجلدات معالجة البيانات (Data Pipeline)

<div align="center">
   
   ```
   txtData/  ──►  foldersData/  ──►  data/  ──►  الموقع
     (خام)          (وسيطة)          (JSON)     (يقرأ منه)
   ```
</div>

<div dir="rtl" align="right">

   يُشغَّل هذا الخط بالكامل بأمر واحد عبر `RUN.sh`.

   **مجلد `txtData` (البيانات الخام):** المجلد الذي يتم وضع بيانات الكلية الخام فيه بصيغة `.txt`.

   **مجلد `codePyData`:** يحتوي على السكريبتات المسؤولة عن تنظيف البيانات وتحويلها:

   | الملف              | الوظيفة                                                                                               |
   | ------------------ | ----------------------------------------------------------------------------------------------------- |
   | `RUN.sh`           | **أهم ملف** — ملف تنفيذي يقوم بتشغيل جميع سكريبتات البايثون بالترتيب الصحيح لتحديث النظام بضغطة واحدة |
   | `foldersOutput.py` | يقرأ الملفات الخام ويقسم الطلاب في لجان امتحانات ويحفظها في مجلد `foldersData`                       |
   | `jsonOutput.py`    | يقرأ اللجان المقسمة ويحولها إلى قواعد بيانات JSON جاهزة لعمل الموقع                                   |
   | `rooms.py`         | يحسب ويوزع عدد الطلاب واللجان لكل مقرر ويحفظها في `txtData/rooms.txt`                                 |
   | `subjectName.py`   | يستخرج أسماء المواد من النصوص وينشئ قائمة بها ويحفظها في `txtData/subjectsName.txt`                   |
   | `AllCode.py`       | يدمج الملفات النصية المختلفة ويحفظها في ملف واحد `txtData/All.txt`                                    |
   | `codeGenLogs.py`   | سكريبت لتوليد سجلات زيارات وهمية (Logs) لاختبار لوحة الإحصائيات ويحفظها في `counterFiles/logs`       |

   **مجلد `foldersData` (بيانات وسيطة):** مجلد يتم إنشاؤه آلياً بواسطة البايثون ويحتوي على لجان الامتحانات مقسمة كملفات نصية قبل تحويلها للوجهة النهائية.

   **مجلد `data` (قاعدة البيانات النهائية):** المجلد الذي يقرأ منه الموقع. يتم توليد هذه الملفات آلياً بالكامل.

   | الملف           | المحتوى                                                  |
   | --------------- | -------------------------------------------------------- |
   | `students.json` | بيانات الطلاب والمقررات المسجلين بها                     |
   | `courses.json`  | تفاصيل المقررات (الاسم، اليوم، التاريخ، الفترة، القاعات) |
   | `exams.json`    | توزيع لجان الامتحانات                                    |
   | `rooms.json`    | قائمة القاعات                                            |

   ### 2. الواجهة الأمامية (Frontend)

   | الملف               | الوظيفة                                                                                                                       |
   | ------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
   | `index.php`         | الصفحة الرئيسية — شريط البحث بالاسم أو الرقم، عرض لجنة الطالب والمادة والقاعة والتاريخ، مع زر لتصدير النتيجة كصورة PNG        |
   | `courses.html`      | صفحة الخطة الدراسية — عرض جميع المقررات مع فلاتر للأقسام، وإمكانية تحديد المواد المكتملة والمتبقية                            |
   | `gpa.html`          | حاسبة المعدل التراكمي التفاعلية                                                                                               |
   | `qa.html`           | الأسئلة الشائعة والروابط الهامة للطلاب                                                                                        |
   | `stats.php`         | لوحة الإحصائيات — رسوم بيانية بـ Chart.js لنشاط الزوار عبر فترات متعددة (ساعة / 6 ساعات / 24 ساعة / 7 أيام / 30 يوم / 12 شهر) |
   | `tour-guide.js`     | جولة تعريفية تظهر للمستخدم عند فتح الموقع لأول مرة، تشرح الميزات خطوة بخطوة                                                   |
   | `dashboardOld.html` | نسخة قديمة من لوحة الإحصائيات كانت تعتمد على Plotly — محفوظة كمرجع أرشيفي فقط                                                 |

   ### 3. الواجهة الخلفية والخادم (Backend)

   | الملف        | الوظيفة                                                                                                                |
   | ------------ | ---------------------------------------------------------------------------------------------------------------------- |
   | `search.php` | نقطة النهاية (API) التي تستقبل طلبات البحث من `index.php` وتجلب البيانات من ملفات JSON                                 |
   | `router.php` | يحاكي عمل `.htaccess` مخصص لتشغيل الموقع بشكل صحيح على خادم PHP المدمج (Local Built-in Server) وإخفاء امتدادات الملفات |

   **مجلد `counterFiles` (نظام الإحصائيات):**

   | الملف             | الوظيفة                                                        |
   | ----------------- | -------------------------------------------------------------- |
   | `counter.php`     | الـ API الخاص بتسجيل الزيارات                                  |
   | `counter-lib.php` | مكتبة دوال العداد (قراءة وكتابة السجلات)                       |
   | `logs`           | المجلد الذي تُحفظ فيه ملفات السجلات (`.jsonl`) لكل صفحة        |
   | `stats_cache`    | مجلد ذاكرة التخزين المؤقت (Cache) لتسريع تحميل صفحة الإحصائيات |

   ### 4. أدوات إضافية (Tools)

   | الملف                          | الوظيفة                                                             |
   | ------------------------------ | ------------------------------------------------------------------- |
   | `getInfo/getStdJson.py`        | أداة CLI للبحث السريع عن طالب في ملفات JSON دون الحاجة لفتح المتصفح |
   | `getInfo/getStdfoldersData.py` | أداة CLI لاختبار سلامة البيانات في المخرجات الوسيطة                 |
   | `images`                      | أيقونات الموقع وشعارات الكلية وصور أماكن اللجان                     |

   ## كيفية تحديث البيانات (Data Workflow)

   تحديث بيانات الموقع للفصل الدراسي الجديد عملية سهلة جداً:

   1- قم بوضع ملفات أقسام الطلاب الخام `.txt` في مجلد `txtData`.

   2- يجب مراجعة ملفات الـ txt للتأكد من أن البيانات مكتوبة بالتنسيق المطلوب المبرمج في النظام

   3- افتح سطر الأوامر (Terminal) وقم بتشغيل ملف الـ Shell:

</div>

   ```bash
   bash codePyData/RUN.sh
   ```
<div dir="rtl" align="right">

   4- سيقوم السكريبت بتنظيف الملفات القديمة، وتقسيم اللجان، وتوليد ملفات `JSON` جديدة تماماً في مجلد `data/`. الموقع الآن محدث!

   ## طريقة التشغيل (محلياً للمطورين)

   1- **استنساخ المستودع (Clone):**

   ```bash
   git clone https://github.com/Ali-Ashraf-Saad/StudentCoursesSearchSystemV2.git
   cd StudentCoursesSearchSystemV2
   ```

   2- **تجهيز البيانات:**

   ```bash
   bash codePyData/RUN.sh
   ```

   3- **تشغيل خادم PHP المحلي (يجب استخدام `router.php`):**

   ```bash
   php -S localhost:8000 router.php
   ```

   4- **الوصول للموقع:**
   افتح المتصفح وانتقل إلى: `http://localhost:8000`
